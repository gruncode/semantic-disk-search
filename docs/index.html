Quick Start

dsearch "query"                                    # colored terminal (50 results)
dsearch -w "query"                                 # sortable HTML table in browser
dsearch -CLexpand-CLrank-CLanswer "query"           # AI answer with citations (~4 min)
dsearch -CLexpand-CLrank-CLanswer -deep "query"     # deep multi-hop search (~7 min)
dsearch -h                                          # all flags

Answer (-CLexpand-CLrank-CLanswer)

  • Query understanding → vector search → 10 agents rank → Claude reads top 15 → cited answer
  • -deep: multi-hop loop discovers terms from results, searches again (max 3 loops)

Step 1: Query Understanding

CLAUDE ▶

One claude -p call parses user text into structured JSON:

{
  "core_query":           "clean search terms (no instructions)",
  "variants":             ["Greek morphology variants + English"],
  "subject_terms":        ["related technical topics"],
  "entities":             ["Intracom", "corpus", "ΕΔΣ"],
  "hypothetical_answer":  "HyDE: fake answer using domain vocabulary"
}
Why separate core_query? If user writes "ψαξε και βρες τους ποιο βασιμους", Cohere embeds "ψαξε" as content — polluting the vector. Claude strips instructions for search, passes them unchanged to agents.

HyDE (Hypothetical Document Embedding)

Claude generates a fake answer using vocabulary the real documents would use. Even a wrong guess contains domain terms that pull in files the query can't reach.

Rebuild from Scratch

6 STEPS ▶
  1. Reindex Recoll — recollindex -c ~/.recoll
  2. Export file list — recollq all MIME types
  3. Extract + chunk — python3 scripts/extract_hybrid.py (6.5 min)
  4. Embed — python3 scripts/cohere_embed2.py (~3.5h, ~$19)
  5. Build ChromaDB — python3 scripts/build_chroma_from_embeddings.py
  6. Install — sudo cp scripts/dsearch /usr/local/bin/
Prerequisites: Recoll 1.34+, Python 3.11+, Cohere API key in ~/.config/dsearch/.env, venv at $VENV_GTE/
